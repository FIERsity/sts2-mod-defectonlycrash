# AGENTS.md

This repository is a reusable Slay the Spire 2 mod development base.

## Purpose

The repository currently includes an example cheat feature set with three
implemented behaviors:

- Click top bar HP to fully heal.
- Click the combat energy orb to refill energy.
- Delete the currently inspected card from the inspect screen via the `删牌`
  button.

## Read First

When starting work in this repo, read these files first:

1. `README.md`
2. `ModTemplate.csproj`
3. `src/Bootstrap/MainFile.cs`
4. `src/Features/Cheats/CheatOverlay.cs`
5. `mod_manifest.json`

## Current Architecture

- `src/Bootstrap/MainFile.cs` is the mod initializer.
- The initializer installs a Godot `CanvasLayer` overlay into the live scene.
- `src/Features/Cheats/CheatOverlay.cs` is the main runtime behavior file.
- The current mod avoids Harmony-based runtime UI patching on macOS because
  Harmony patch application was failing at runtime with
  `NotImplementedException` in this environment.

If you need to add new cheat-style UI, prefer extending the overlay first.

## Build Assumptions

- Target stack: `Godot 4.5.1`, `.NET 9`, `C#`
- The game assemblies are loaded from the local Slay the Spire 2 install.
- Local machine overrides belong in `ModTemplate.local.props`, not in
  `ModTemplate.csproj`.

Typical commands:

```bash
dotnet build
dotnet publish
```

## macOS-Specific Rule

On macOS, the game scans mods from:

`SlayTheSpire2.app/Contents/MacOS/mods`

Do not switch this back to `Contents/Resources/mods` unless you have reverified
 the game's loader behavior.

## Local File Policy

- `ModTemplate.local.props` is machine-local and should stay untracked.
- `packages/` is generated by local restore and should stay untracked.
- `.godot/` is generated and should stay untracked.

## Editing Guidance

- Prefer keeping gameplay logic in a small number of files.
- If adding new runtime UI, keep it inside `src/Features/` and split by feature
  instead of growing one giant file forever.
- Keep `src/Bootstrap/` small. It should stay as entrypoint wiring, not feature
  implementation.
- If adding real content later, create dedicated folders under `src/Features/`
  or `content/` rather than expanding the bootstrap file.

## If You Revisit Harmony

Before reintroducing Harmony patches:

1. Confirm the current game version and runtime behavior.
2. Verify that `PatchAll` works on the target platform.
3. Check whether the same goal can be achieved by scene overlay or direct node
   interaction instead.

Do not assume a working Windows Harmony approach will behave the same on macOS.

## Repo Identity vs Live Mod Identity

Keep these separate:

- The repository should stay generic and reusable.
- A built mod copied into the game's `mods` directory may use a different final
  name, author, version, and manifest.

Do not rename the repository itself to match one shipped mod unless explicitly
requested.
